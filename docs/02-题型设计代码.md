# 02 · 题型设计代码

> 全部示例使用**自编占位内容**(咖啡、风车、灯塔等通用主题),不引用真实考试题库文本。Developed by **Jimmy**

本文档系统性列出 IELTS 阅读练习系统支持的 **11 种题型**,每种题型包含:

1. **Schema 字段**:数据 JSON 里 `questionGroups` 一项的结构
2. **HTML 模板**:渲染到练习页面的 `bodyHtml` 片段
3. **用户交互**:点击 / 拖拽 / 输入等行为
4. **答案采集**:`collectAnswers()` 如何读出用户答案
5. **答案 Key**:`answerKey` 里的写法约定
6. **评分逻辑**:`AnswerMatchCore.compareAnswers` 比对规则

---

## 0. 顶层数据结构

每篇阅读题 = 一个 `.js` 文件,在启动时通过 `__READING_EXAM_DATA__.register(examId, data)` 注册:

```js
{
    "schemaVersion": "ReadingExamSourceV1",
    "examId": "p1-custom-001",
    "meta": {
        "title": "Coffee Cultivation 占位标题",
        "category": "P1",                 // P1 / P2 / P3
        "frequency": "high",              // high / medium / low
        "pdfFilename": "可选, 用于侧边 PDF 链接",
        "questionIntroHtml": "<h3>Questions</h3>"
    },
    "passage": {
        "blocks": [
            { "blockId": "passage-main", "kind": "html", "html": "<h2>…</h2>" }
        ]
    },
    "questionGroups": [ /* 见后文 */ ],
    "answerKey": { "q1": "A", "q2": ["coffee", "tea"], "q3": "true" },
    "questionOrder": ["q1", "q2", "q3"],
    "questionDisplayMap": { "q1": "1", "q2": "2", "q3": "3" }
}
```

**关键约定**:

- `questionGroups[i].kind` 是 11 种之一(见目录)
- 每个 `questionGroups[i].bodyHtml` 内部所有交互控件必须挂 `data-question="qN"` 属性(`name` 也可以,用于 radio/checkbox)
- `answerKey[qN]` 的写法因 kind 不同而不同,详见每节

---

## 目录

| # | Kind | 中文 | 输入类型 |
|---|------|------|---------|
| 1 | `single_choice` | 单选 | radio |
| 2 | `multi_choice` | 多选 | checkbox |
| 3 | `true_false_not_given` | T/F/NG 判断 | radio × 3 |
| 4 | `yes_no_not_given` | Y/N/NG 观点判断 | radio × 3 |
| 5 | `matching` | 匹配(可拖拽 / 下拉) | drop / select |
| 6 | `classification` | 分类 | drop / select(可重用) |
| 7 | `summary_completion` | 摘要填空 | text input |
| 8 | `sentence_completion` | 句子填空 | text input |
| 9 | `table_completion` | 表格填空 | text input in table cells |
| 10 | `diagram_completion` | 图示填空 | text input + 背景图 |
| 11 | `short_answer` | 简答题 | text input |

## 1. single_choice — 单选

**说明**:经典四选一(也常见三选一),一题一组 radio,题号绑定到 `name`。

### Schema

```json
{
    "groupId": "group-q14-17",
    "kind": "single_choice",
    "questionIds": ["q14", "q15", "q16", "q17"],
    "bodyHtml": "<!-- 见下 -->"
}
```

### HTML 模板

```html
<div class="group">
    <h4>Questions 14–17</h4>
    <p>Choose the correct letter, <strong>A</strong>, <strong>B</strong>, <strong>C</strong> or <strong>D</strong>.</p>

    <div class="question-item" id="q14-anchor">
        <p><strong>14</strong> 占位题干:咖啡最早被发现的地点。</p>
        <ul class="choice-list">
            <li><label><input type="radio" name="q14" value="A"> A 占位选项一</label></li>
            <li><label><input type="radio" name="q14" value="B"> B 占位选项二</label></li>
            <li><label><input type="radio" name="q14" value="C"> C 占位选项三</label></li>
            <li><label><input type="radio" name="q14" value="D"> D 占位选项四</label></li>
        </ul>
    </div>
</div>
```

### 答案采集

```js
// unifiedReadingPage.js: collectAnswers()
const radios = document.querySelectorAll('input[type="radio"][name="q14"]');
const checked = Array.from(radios).find(r => r.checked);
answers["q14"] = checked ? checked.value : "";
```

### Answer Key

```json
"answerKey": { "q14": "A", "q15": "C", "q16": "B", "q17": "D" }
```

### 评分规则

- 大小写不敏感(`a` ≈ `A`)
- 单字母直接 token 比对,不做语义匹配

## 2. multi_choice — 多选

**说明**:N 选 K(常见 5 选 2 / 7 选 3),多个 checkbox 共用同一组 `name`。

### Schema

```json
{
    "groupId": "group-q18-19",
    "kind": "multi_choice",
    "questionIds": ["q18", "q19"],
    "bodyHtml": "<!-- 见下 -->"
}
```

### HTML 模板

```html
<div class="group">
    <h4>Questions 18–19</h4>
    <p>Choose <strong>TWO</strong> letters, <strong>A–E</strong>.</p>
    <p>Which TWO factors led to the company's growth?</p>

    <ul class="choice-list">
        <li><label><input type="checkbox" name="q18-19" value="A"> A 占位因素 1</label></li>
        <li><label><input type="checkbox" name="q18-19" value="B"> B 占位因素 2</label></li>
        <li><label><input type="checkbox" name="q18-19" value="C"> C 占位因素 3</label></li>
        <li><label><input type="checkbox" name="q18-19" value="D"> D 占位因素 4</label></li>
        <li><label><input type="checkbox" name="q18-19" value="E"> E 占位因素 5</label></li>
    </ul>
</div>
```

> 注意 `name="q18-19"`(连号表示这组题共享一份 checkbox 列表)。

### 答案采集

```js
// collectAnswers() 内有 getCheckboxAnswers() 专门处理
const boxes = document.querySelectorAll('input[type="checkbox"][name="q18-19"]:checked');
const picks = Array.from(boxes).map(b => b.value).sort();   // 例如 ["A", "C"]
// 然后 expandQuestionSequence("q18-19") = ["q18", "q19"]
// answers["q18"] = picks[0]; answers["q19"] = picks[1];
```

### Answer Key

```json
"answerKey": {
    "q18": "A",
    "q19": "C"
}
```

也可以两个题号都用同一个集合答案:

```json
"answerKey": {
    "q18": ["A", "C"],
    "q19": ["A", "C"]
}
```

### 评分规则

- 用户实际勾选的字母按 ASCII 排序后**与 answerKey 按题号 1:1 对齐**
- 多余/缺少都算错

## 3. true_false_not_given — T/F/NG

**说明**:对客观事实的判断。

### Schema

```json
{
    "groupId": "group-q20-23",
    "kind": "true_false_not_given",
    "questionIds": ["q20", "q21", "q22", "q23"],
    "bodyHtml": "<!-- 见下 -->"
}
```

### HTML 模板

```html
<div class="group">
    <h4>Questions 20–23</h4>
    <p>Do the following statements agree with the information given in the passage?</p>
    <ul>
        <li><strong>TRUE</strong> if the statement agrees with the information</li>
        <li><strong>FALSE</strong> if the statement contradicts the information</li>
        <li><strong>NOT GIVEN</strong> if there is no information on this</li>
    </ul>

    <div class="question-item" id="q20-anchor">
        <p><strong>20</strong> 占位陈述:风车在 14 世纪首次出现于尼德兰。</p>
        <label><input type="radio" name="q20" value="TRUE"> TRUE</label>
        <label><input type="radio" name="q20" value="FALSE"> FALSE</label>
        <label><input type="radio" name="q20" value="NOT GIVEN"> NOT GIVEN</label>
    </div>
    <!-- q21, q22, q23 同上 -->
</div>
```

### Answer Key

```json
"answerKey": {
    "q20": "TRUE",
    "q21": "FALSE",
    "q22": "NOT GIVEN",
    "q23": "TRUE"
}
```

### 评分规则(关键)

`AnswerMatchCore.normalizeToken` 会把以下值归一化:

| 用户输入 | 归一化后 |
| -------- | -------- |
| `TRUE` / `True` / `true` / `T` / `t` | `true` |
| `YES` / `Y` / `Yes` | `true` |
| `FALSE` / `f` / `False` | `false` |
| `NO` / `N` / `no` | `false` |
| `NG` / `notgiven` / `not-given` | `not given` |

→ 所以 `T/F/NG` 与 `Y/N/NG` 互相兼容,**写错也能识别**。这是优点也是潜在坑(见 [04-同题型不同操作逻辑.md](04-同题型不同操作逻辑.md))。

## 4. yes_no_not_given — Y/N/NG

**说明**:与 T/F/NG 几乎一致,但针对作者**观点**而不是事实。HTML 把 TRUE/FALSE 改成 YES/NO 即可。

### HTML 模板差异

```html
<label><input type="radio" name="q24" value="YES"> YES</label>
<label><input type="radio" name="q24" value="NO"> NO</label>
<label><input type="radio" name="q24" value="NOT GIVEN"> NOT GIVEN</label>
```

### Answer Key

```json
"answerKey": {
    "q24": "YES",
    "q25": "NO",
    "q26": "NOT GIVEN"
}
```

> 由于归一化时 `YES → true`、`NO → false`,你也可以全部写成 `TRUE/FALSE`,系统仍能正确匹配。推荐保留原始字面以方便对错回放。

## 5. matching — 匹配

**说明**:把一组项匹配到另一组(常见 heading→paragraph、statement→country/person)。

### Schema

```json
{
    "groupId": "group-q1-8",
    "kind": "matching",
    "questionIds": ["q1", "q2", "q3", "q4", "q5", "q6", "q7", "q8"],
    "bodyHtml": "<!-- 见下 -->",
    "allowOptionReuse": false       // true 表示同一 heading 可重复使用
}
```

### HTML 模板(两种主流写法)

#### 写法 A:拖拽到段落上的 dropzone

```html
<div class="paragraph-wrapper" id="q1-anchor">
    <div class="paragraph-dropzone dropzone" data-paragraph="A" data-question="q1">
        <span class="paragraph-label">Paragraph A (Q1):</span>
        <div class="dropped-items"></div>
    </div>
    <p><strong>A</strong> 占位段落正文…</p>
</div>

<div class="headings-pool" id="headings-pool-container">
    <strong>List of Headings</strong>
    <div class="pool-items">
        <div class="drag-item" draggable="true" data-heading="i">i. A chance discovery</div>
        <div class="drag-item" draggable="true" data-heading="ii">ii. A new technology emerges</div>
        <!-- ... -->
    </div>
</div>
```

#### 写法 B:dropzone 集中在题目列表里

```html
<div class="match-question-item" id="q9-anchor">
    <div class="match-dropzone" data-question="q9"></div>
    <p><strong>9</strong> 占位陈述:谁最早提出了风能商业化方案?</p>
</div>

<div class="options-pool">
    <strong>List of People</strong>
    <div class="pool-items" style="flex-direction: column;">
        <div class="drag-item" draggable="true" data-option="A">A 占位姓名一</div>
        <div class="drag-item" draggable="true" data-option="B">B 占位姓名二</div>
        <!-- ... -->
    </div>
</div>
```

→ 同一 kind,两种 DOM,详见 [04-同题型不同操作逻辑.md](04-同题型不同操作逻辑.md)。

### 答案采集

```js
// getDropzoneAnswer(qN): 优先匹配 .paragraph-dropzone[data-question]
//                       其次 .match-dropzone[data-question]
//                       回退 .dropzone[data-question]
const zone = document.querySelector(`.match-dropzone[data-question="q9"], .paragraph-dropzone[data-question="q9"]`);
const dropped = zone.querySelector('.drag-item');
answers["q9"] = dropped ? dropped.dataset.option || dropped.dataset.heading : "";
```

### Answer Key

```json
"answerKey": {
    "q1": "viii",     // 罗马数字 heading
    "q2": "iv",
    "q9": "D",        // 字母选项
    "q10": "E"
}
```

### 评分规则

- 罗马数字 / 字母大小写不敏感
- 但 **单字母与多字母拼写不混淆**(`A` 不等于 `a long answer`)

## 6. classification — 分类

**说明**:与 matching 类似,但**选项可重复使用**(`allowOptionReuse: true`),且分类池一般只有 3–5 个,要分给 ≥6 个项目。

### Schema

```json
{
    "groupId": "group-classify",
    "kind": "classification",
    "questionIds": ["q1", "q2", "q3", "q4", "q5", "q6"],
    "bodyHtml": "<!-- 见下 -->",
    "allowOptionReuse": true
}
```

### HTML 模板(沿用 matching 写法 B 即可)

```html
<p>Classify the following plants as occurring in:</p>
<p><strong>A</strong> high mountain regions<br>
   <strong>B</strong> tropical lowlands<br>
   <strong>C</strong> coastal areas</p>

<div class="match-question-item" id="q1-anchor">
    <div class="match-dropzone" data-question="q1"></div>
    <p><strong>1</strong> 占位项目:Plant Alpha</p>
</div>
<!-- q2..q6 同上 -->

<div class="options-pool">
    <div class="pool-items">
        <div class="drag-item" draggable="true" data-option="A">A high mountain</div>
        <div class="drag-item" draggable="true" data-option="B">B tropical lowlands</div>
        <div class="drag-item" draggable="true" data-option="C">C coastal</div>
    </div>
</div>
```

### Answer Key

```json
"answerKey": {
    "q1": "A", "q2": "B", "q3": "A", "q4": "C", "q5": "B", "q6": "A"
}
```

### 评分规则

与 matching 一致,但前端代码会在 `data-allow-reuse="true"` 时**不消耗选项池**(用户拖出一个分类后,池里仍保留)。

## 7. summary_completion — 摘要填空

**说明**:一段总结性段落,中间多个空格,可能附带"词库"(box of words)。

### Schema

```json
{
    "groupId": "group-summary",
    "kind": "summary_completion",
    "questionIds": ["q27", "q28", "q29", "q30"],
    "bodyHtml": "<!-- 见下 -->"
}
```

### HTML 模板(无词库 / 直接填空)

```html
<div class="group">
    <h4>Questions 27–30</h4>
    <p>Complete the summary using <strong>NO MORE THAN TWO WORDS</strong> from the passage.</p>

    <div class="summary-paragraph">
        <p>
            The 占位主题 was first introduced in
            <input type="text" data-question="q27" class="answer-input" maxlength="40" />
            and gained popularity through
            <input type="text" data-question="q28" class="answer-input" maxlength="40" />.
            Today it remains the most common form of
            <input type="text" data-question="q29" class="answer-input" maxlength="40" />,
            especially in
            <input type="text" data-question="q30" class="answer-input" maxlength="40" />.
        </p>
    </div>
</div>
```

### HTML 模板(有词库 / drop 进段落)

```html
<div class="word-bank">
    <strong>List of Words</strong>
    <div class="pool-items">
        <div class="drag-item" draggable="true" data-word="industry">industry</div>
        <div class="drag-item" draggable="true" data-word="harvest">harvest</div>
        <div class="drag-item" draggable="true" data-word="trade">trade</div>
        <!-- ... -->
    </div>
</div>

<p>
    The crop was first cultivated for
    <span class="drop-target-summary dropzone" data-question="q27"></span>,
    and later became a popular item of
    <span class="drop-target-summary dropzone" data-question="q28"></span>.
</p>
```

### 答案采集

```js
// 两种都被 collectAnswers() 覆盖:
// 1. textarea/input → getTextualAnswer(qN) 读 value
// 2. .drop-target-summary[data-question] → getDropzoneAnswer(qN) 读拖入的 data-word
```

### Answer Key

```json
"answerKey": {
    "q27": "harvest",
    "q28": "trade",
    "q29": ["industry", "industries"]    // 允许复数变体
}
```

### 评分规则

- 字符串归一化:去除标点 / 双引号 / 大小写
- 用空格 / 逗号 / 斜杠分隔多个 token
- 如果 `answerKey` 是数组,**任一**命中即对

## 答案 Key 通用规则汇总

| 类别 | 写法 | 评分行为 |
| ---- | ---- | -------- |
| 单一答案 | `"q1": "A"` | 字符串归一化后比对 |
| 多接受写法 | `"q1": ["coffee", "tea"]` | 任一命中即对 |
| 多空合一 | `"q1": ["A", "C"]` 配合 `q1 = A,C` 输入 | 集合比对 |
| 大小写 | 自动忽略(单字母除外) | A==a 但 "Apple" ≠ "apple"…(实际也 ≈,因 normalizeLooseToken 兜底) |
| 标点 | 头尾标点会被剥离 | "harvest." → "harvest" |
| 引号 | 弯引号 → 直引号 | "café" 不变 |
| 连字符变体 | `– — ‐` → `-` | "co-operate" 统一 |
| 同义判断词 | T=YES=true,F=NO=false,NG=NOT GIVEN | 自动等价 |

## 附录:bodyHtml 的样式与脚本规则

1. **只用项目内已有 CSS class**(`group / question-item / drag-item / dropzone / pool-items / answer-input` 等)
2. **不要写 inline `<script>`**,所有交互都由 `unifiedReadingPage.js` 接管
3. **必须挂 `data-question="qN"`** 或 `name="qN"`(radio/checkbox 用 name)
4. **anchor id 命名**:`<div id="qN-anchor">` 用于题目跳转条
5. **多空合一**(checkbox 多选)用 `name="qN-qM"` 形式

完整的 class 命名约定见 [css/main.css](../css/main.css) 中的 `.group`、`.question-item`、`.answer-input`、`.match-dropzone`、`.drag-item`、`.pool-items` 等定义。
