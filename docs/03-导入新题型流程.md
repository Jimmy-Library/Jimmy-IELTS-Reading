# 03 · 导入新题型流程

> 三条路径,按"改动量从小到大"排列。Developed by **Jimmy**

| 场景 | 路径 | 难度 | 适合人群 |
| ---- | ---- | ---- | -------- |
| 已有 11 种题型,只想新增一篇题目 | **路径 A** | ★ | 任何人 |
| 已有题型,但 HTML 模板要小幅自定义(布局、配图) | **路径 B** | ★★ | 会复制粘贴 |
| 真正需要"新题型"(系统没有任何相似 kind) | **路径 C** | ★★★★ | 前端工程师 |

---

## 路径 A:用编辑器生成新题目(推荐)

### 步骤

1. 浏览器打开 [tools/reading-editor.html](tools/reading-editor.html)
2. 顶部填基础信息:
   - **examId**:必须**唯一**且**全小写**,推荐格式 `p1-custom-001` / `p2-jimmy-thanksgiving` / `mock1-r1`
   - **title**:中英任意
   - **category**:`P1` / `P2` / `P3`
   - **frequency**:`high` / `medium` / `low`(显示用,不影响评分)
3. **文章** 标签页:粘贴或键入文章正文(支持简单 HTML;留空换行即可分段)
4. **题组** 标签页:点 `+ 添加题组`,在题型下拉里选合适 kind,按表单填空
5. 实时检查 **预览** 标签:看到与正式练习页几乎一致的渲染
6. **导出** 标签 → 选 `ReadingExamSourceV1 JSON (.js)`
7. 把下载的 `.js` 拖进 `assets/generated/reading-exams/`
8. 打开 `assets/generated/reading-exams/manifest.js`,在最外层对象**任意位置**新增一项:

   ```js
   "p1-custom-001": {
       "examId":   "p1-custom-001",
       "dataKey":  "p1-custom-001",
       "script":   "./p1-custom-001.js",
       "title":    "你的自定义标题",
       "category": "P1"
   },
   ```

   > `dataKey` 通常和 `examId` 相同;`script` 是相对路径,保持 `./` 前缀。
9. 浏览器刷新 → 设置 → 🔄 强制刷新题库 → 题库浏览里就能找到新题

### 编辑器导出格式说明

编辑器同时支持 3 种导出:

| 格式 | 用途 |
| ---- | ---- |
| `.json` | 纯数据,可二次处理 / 走 schema 校验 |
| `.js` | 已包好 `(function registerReadingExamData...)`,**直接放进项目即可** |
| `.html` | 独立可运行的单页练习页(不依赖项目),方便分享给他人单独试做 |

## 路径 B:手写已有 kind 的题目

适合**编辑器实现不了的细节**(如自定义图、注释、分栏)。

### 步骤

1. 复制现有相近题目作为模板:
   ```powershell
   Copy-Item assets\generated\reading-exams\p1-high-01.js `
             assets\generated\reading-exams\p1-custom-002.js
   ```
2. 用编辑器打开新文件,把:
   - `register("p1-high-01", ...)` 改成 `register("p1-custom-002", ...)`
   - `examId` 改成 `"p1-custom-002"`
   - `meta.title` 改新
   - `passage.blocks[0].html` 替换为新的文章正文
   - `questionGroups[i].bodyHtml` 替换为新题组(保持 `data-question="qN"` 不变)
   - `answerKey` 全部重写
   - `questionOrder` / `questionDisplayMap` 与新题数对齐
3. manifest.js 新增条目(同路径 A 步骤 8)
4. 刷新,验证

### 编写题组 HTML 时的硬性约定

| 约定 | 例 | 后果 |
| ---- | ---- | ---- |
| 控件必须有 `data-question="qN"` 或 `name="qN"` | `<input data-question="q14">` | 没有 → 答案永远是空 |
| radio / checkbox 共享 `name` | `<input type="radio" name="q14">` | 不共享 → 多选会失效 |
| 拖拽源用 class `drag-item` + `data-heading` / `data-option` / `data-word` | `<div class="drag-item" draggable="true" data-heading="iv">…</div>` | 缺 class → 不能拖 |
| 拖拽目标用 class `dropzone`(或 `paragraph-dropzone` / `match-dropzone` / `drop-target-summary`) | `<div class="match-dropzone" data-question="q9">` | 缺 class → 不接收拖入 |
| 文本框统一用 `class="answer-input"` | `<input class="answer-input">` | 不影响功能,影响样式 |
| 锚点 id 用 `qN-anchor` | `<div id="q14-anchor">` | 不影响功能,影响题号导航 |

### 常见小自定义示例

#### B-1. 在选项里嵌入小图标

```html
<li>
    <label>
        <input type="radio" name="q14" value="A">
        A <img src="assets/diagrams/leaf-icon.svg" width="16" alt=""> 占位选项
    </label>
</li>
```

→ 不影响评分,图片要放在项目目录里,**不要外链**。

#### B-2. summary_completion 加段间标题

```html
<div class="summary-paragraph">
    <h5>Origins</h5>
    <p>The 占位主题 was first described in
        <input type="text" data-question="q27" class="answer-input" />…
    </p>
    <h5>Modern Use</h5>
    <p>By the year <input type="text" data-question="q28" class="answer-input" />,…</p>
</div>
```

#### B-3. matching 改成下拉框(替代拖拽)

部分用户在触屏设备上更喜欢下拉:

```html
<div class="match-question-item">
    <p><strong>9</strong> 占位陈述</p>
    <select data-question="q9" class="answer-select">
        <option value="">— 选择 —</option>
        <option value="A">A 占位选项一</option>
        <option value="B">B 占位选项二</option>
    </select>
</div>
```

`collectAnswers` 会通过 `getTextualAnswer(qN)` 自动读到 `select.value`。

## 路径 C:新增一种全新题型(kind)

仅当 11 种现有 kind 都不合用时才走这条路。**改动会涉及核心代码**,需要前端能力。

### Step 1. 给新 kind 起名

命名约定:**小写下划线**,语义清晰。例如:
- `flow_chart_completion`(流程图填空)
- `picture_choice`(看图选项)
- `passage_ordering`(段落排序)

### Step 2. 准备题目数据

在 `questionGroups[i]` 里使用新 `kind`:

```js
{
    "groupId": "group-pict",
    "kind": "picture_choice",
    "questionIds": ["q1", "q2", "q3"],
    "bodyHtml": "<!-- 自己的 HTML -->",
    "allowOptionReuse": false
}
```

### Step 3. 写 bodyHtml(交互骨架)

**核心要求**:控件必须有 `data-question="qN"` 或者 `name="qN"`,让 `collectAnswers()` 能采到。

```html
<div class="group" data-kind="picture_choice">
    <h4>Questions 1–3</h4>
    <p>Choose the matching diagram for each statement.</p>

    <div class="question-item" id="q1-anchor">
        <p><strong>1</strong> 占位陈述</p>
        <div class="picture-options">
            <label><input type="radio" name="q1" value="A">
                <img src="assets/diagrams/pic-1a.svg" alt="A"></label>
            <label><input type="radio" name="q1" value="B">
                <img src="assets/diagrams/pic-1b.svg" alt="B"></label>
            <label><input type="radio" name="q1" value="C">
                <img src="assets/diagrams/pic-1c.svg" alt="C"></label>
        </div>
    </div>
</div>
```

### Step 4. 让 `unifiedReadingPage.js` 知道这个 kind 也走"选项分屏"

打开 [js/runtime/unifiedReadingPage.js](../js/runtime/unifiedReadingPage.js),搜索:

```js
const EXPLANATION_SPLIT_KINDS = new Set([
    'single_choice',
    'multi_choice',
    'true_false_not_given',
    'yes_no_not_given'
]);
```

把新 kind 加进去(如果它使用 radio / checkbox):

```js
const EXPLANATION_SPLIT_KINDS = new Set([
    'single_choice',
    'multi_choice',
    'true_false_not_given',
    'yes_no_not_given',
    'picture_choice'              // ← 新增
]);
```

> 这个集合用于讲解模式的"左右分屏"渲染逻辑;漏掉的 kind 不会影响答题,只是讲解界面会一栏到底。

### Step 5. 答案采集兼容性

不需要再改了,只要 Step 3 的 HTML 控件挂了 `data-question` / `name`,系统会自动:

- `input[type=radio][name="qN"]` → 读 `value`
- `input[type=checkbox][name="qN"]` → 集合形式
- `[data-question="qN"]` 上的 `value` / `textContent` / 拖入项 → 文本采集
- `.dropzone[data-question="qN"]` 里嵌的 `.drag-item` → 读 `data-heading` / `data-option` / `data-word`

### Step 6. answerKey 与评分

只要 answerKey 写规范字符串,`AnswerMatchCore.compareAnswers` 就能用:

```json
"answerKey": {
    "q1": "B",
    "q2": "A",
    "q3": "C"
}
```

若涉及多答案,用数组(详见 [02-题型设计代码.md](02-题型设计代码.md) 第 11 节末附录)。

### Step 7. 在编辑器里添加该 kind(可选)

打开 [tools/reading-editor.html](tools/reading-editor.html),搜索 `KIND_DEFS`(题型定义注册表),按照已有项写一份:

```js
KIND_DEFS.picture_choice = {
    label: '看图选项',
    description: '题干 + 多张图片选项,单选 radio',
    fields: [
        { key: 'instructions', type: 'textarea', label: '题组说明' },
        { key: 'questions', type: 'picture-list', label: '题目与图片' }
    ],
    toBodyHtml(group, helpers) {
        // 把 group.questions 渲染成 HTML 字符串
        // ...
    },
    fromBodyHtml(html) {
        // 反序列化(便于编辑器再次打开同一题目)
        // ...
    }
};
```

> 这一步非必需 → 不在编辑器加,运维同学仍可手写 HTML 后用路径 B 导入。

### Step 8. 测试清单

- [ ] manifest 添加,刷新后能看到新题
- [ ] 点击进入,看到 bodyHtml 正确渲染
- [ ] 答题:radio / checkbox / 拖拽 / 输入框都能交互
- [ ] 提交:console 无报错
- [ ] 阅卷:得分与预期一致(可故意答错验证)
- [ ] 历史:在练习记录里能找到本次答题
- [ ] 回放:点击记录详情 → "回放" 能复现答案
- [ ] PDF / PNG 导出:能正常生成

## 路径 C 实例:添加 `passage_ordering`(段落排序)

下面给一个完整 demo,演示新增题型的全流程。

### 1. 准备一条最小练习数据

```js
// assets/generated/reading-exams/p1-custom-order-001.js
(function registerReadingExamData(global) {
    'use strict';
    global.__READING_EXAM_DATA__.register("p1-custom-order-001", {
        "schemaVersion": "ReadingExamSourceV1",
        "examId": "p1-custom-order-001",
        "meta": {
            "title": "占位 · 段落排序示例",
            "category": "P1",
            "frequency": "high"
        },
        "passage": {
            "blocks": [{
                "blockId": "p", "kind": "html",
                "html": "<h2>READING PASSAGE</h2><p>占位说明:阅读以下打乱的段落,然后排序。</p>"
            }]
        },
        "questionGroups": [{
            "groupId": "g1",
            "kind": "passage_ordering",
            "questionIds": ["q1", "q2", "q3", "q4"],
            "bodyHtml": `
                <div class="group" data-kind="passage_ordering">
                    <h4>Questions 1–4</h4>
                    <p>把段落 A–D 按文章原顺序排列。</p>
                    <div class="match-question-item" id="q1-anchor">
                        <span>第 1 段:</span>
                        <select data-question="q1" class="answer-select">
                            <option value="">—</option>
                            <option value="A">A</option><option value="B">B</option>
                            <option value="C">C</option><option value="D">D</option>
                        </select>
                    </div>
                    <div class="match-question-item" id="q2-anchor">
                        <span>第 2 段:</span>
                        <select data-question="q2" class="answer-select">
                            <option value="">—</option>
                            <option value="A">A</option><option value="B">B</option>
                            <option value="C">C</option><option value="D">D</option>
                        </select>
                    </div>
                    <!-- q3, q4 同上 -->
                </div>
            `,
            "allowOptionReuse": false
        }],
        "answerKey": {
            "q1": "C", "q2": "A", "q3": "D", "q4": "B"
        },
        "questionOrder": ["q1", "q2", "q3", "q4"],
        "questionDisplayMap": { "q1": "1", "q2": "2", "q3": "3", "q4": "4" }
    });
})(typeof window !== "undefined" ? window : globalThis);
```

### 2. 更新 manifest

```js
"p1-custom-order-001": {
    "examId":   "p1-custom-order-001",
    "dataKey":  "p1-custom-order-001",
    "script":   "./p1-custom-order-001.js",
    "title":    "占位 · 段落排序示例",
    "category": "P1"
},
```

### 3. 不需要改任何核心代码

虽然 kind 是新的(`passage_ordering`),但因为我们用的是 `<select data-question>`,`collectAnswers()` 已经能通过 `getTextualAnswer(qN)` 自动采到 `select.value`。`compareAnswers` 也照常工作。

→ **这就是这个项目的设计精髓**:kind 仅是元数据标签,真正决定行为的是 HTML 控件的 attribute。新增 kind 在多数情况下不需要改核心代码,只需要写好 bodyHtml。

---

## 反向操作:删除一种 kind

如果想停用某 kind:

1. 在所有题目 `.js` 里把该 `questionGroups[i].kind` 改成其它接近的(如 `flow_chart_completion` → `summary_completion`)
2. 编辑器里删掉 `KIND_DEFS.<kind>`(可选)

> 不建议从 `EXPLANATION_SPLIT_KINDS` 移除已有 kind,会影响讲解模式的兼容性。

## 总结对照表

| 你想做的事 | 走哪条路径 | 改动文件 |
| -------- | -------- | ------- |
| 加一篇标准题 | A(编辑器) | 1 个 `.js` + manifest.js |
| 改答案 | 直接编辑 `.js` 里的 `answerKey` | 仅该题 `.js` |
| 换段配图 | B(手写) | 仅该题 `.js` |
| 出一篇含 3 种已有题型混合题 | A(编辑器) | 1 个 `.js` + manifest.js |
| 新增"看图选项"题型 | C | 1 个 `.js` + manifest.js + `unifiedReadingPage.js` 一行 |
| 完全移除某题型支持 | 自上而下清理使用方 | 所有引用该 kind 的 `.js` |
