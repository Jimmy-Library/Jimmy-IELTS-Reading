## 2. 启动与停止

### 2.1 单机直接打开(不推荐生产)

双击 `index.html`,首次进入会弹出 GPL 协议说明。

**已知限制**(`file://` 协议):
- 部分浏览器禁止 `webkitdirectory` 读取
- localStorage 跨标签不隔离
- 某些 Chromium 版本会拦截 `postMessage`

→ 推荐使用第 2.2 节的 HTTP 方式。

### 2.2 本地 HTTP 服务器

```powershell
# Windows
cd C:\Users\James\Desktop\网页版机考部署
python -m http.server 8080
# 浏览器访问 http://127.0.0.1:8080/
```

```bash
# macOS / Linux
cd 网页版机考部署
python3 -m http.server 8080
```

### 2.3 局域网部署(IIS / Nginx)

参见 [../README.md 第五节](../README.md#五公司局域网部署教程重点)。

### 2.4 后台启动 + 开机自启(Windows)

`scripts/start-ielts.bat` 示例(自行新建):

```bat
@echo off
cd /d C:\IELTS-CBT
start /b "" python -m http.server 80
exit
```

放到 `shell:startup`(`Win + R` 输入)即可开机自启。

## 3. 数据存储位置

| 数据 | 位置 | 备注 |
| ---- | ---- | ---- |
| 练习记录 | 浏览器 `localStorage["exam_system_practice_records"]` | 用户级,清浏览器数据即丢 |
| 统计汇总 | `localStorage["exam_system_user_stats"]` | 由 `practiceRecorder` 维护 |
| 用户设置 | `localStorage["exam_system_settings"]` | 主题、引导状态等 |
| 备份列表 | `localStorage["exam_system_backups"]` | 限 20 份,FIFO 替换 |
| 题库索引(默认) | `localStorage["exam_index"]` | 由 `manifest.js` 引导 |
| 题库数据(默认) | 内存 `window.__READING_EXAM_DATA__` | 由生成的 .js 在启动时 register |
| 自定义题库 | `localStorage["exam_index_<时间戳>"]` | 多套配置切换 |

**导出全部 localStorage**(浏览器 DevTools `Console`):

```js
copy(JSON.stringify(Object.entries(localStorage)
    .filter(([k]) => k.startsWith('exam_system_'))
    .reduce((acc, [k, v]) => (acc[k] = v, acc), {}), null, 2));
```

剪贴板里的内容就是完整快照,粘贴到 `.json` 文件即可。

## 4. 日常维护任务

### 4.1 清缓存与强制刷新

| 场景 | 操作 |
| ---- | ---- |
| 代码修改后界面没变化 | `Ctrl + Shift + R`(强制刷新) |
| 题目修改后没生效 | 设置 → 🔄 强制刷新题库 |
| localStorage 数据混乱 | 设置 → 🗑️ 清除缓存(保留题库,清统计) |
| 完全重置 | DevTools → Application → Storage → Clear site data |

### 4.2 备份与恢复

**导出备份(管理员视角)**:
1. `📝 练习记录` → 📤 导出数据
2. 下载的 JSON 含 records / stats / 校验和

**恢复**:
1. `📝 练习记录` → 📥 导入数据
2. 选择 JSON,选合并策略:
   - **覆盖**:导入 = 当前(危险)
   - **合并**:已有保留,新条目追加(推荐)
   - **跳过**:仅添加全新条目

**定期备份策略**(企业环境):
- 让所有客户端共享一个 SMB 目录(如 `\\server\backup\ielts\`)
- 用浏览器扩展或 Tampermonkey 脚本每周自动调用 `localStorage` → 文件
- 服务器端 cron 拉一次备份归档

### 4.3 题库管理

| 操作 | 入口 | 适用场景 |
| ---- | ---- | -------- |
| 重新扫描默认题库 | 设置 → 🔄 强制刷新题库 | 数据看起来不对 |
| 加载新文件夹 | 设置 → 📂 加载题库 | 接入老师交付的新题包 |
| 多套配置切换 | 设置 → ⚙️ 题库配置切换 | 同一台机器服务多个班 |
| 删除已加载的题库 | 设置 → 题库配置 → 选中 → 删除 | 学员误导入了不需要的题目 |

### 4.4 主题与界面

绿色风格已经写死,不建议再切换其它主题(原项目的 `晨雾群山 / 深海孤航 / 落日雾花` 已统一指向 `forest-green` 着色器)。

如果硬要改色:
- 全局主色变量在 [css/main.css `:root`](../css/main.css)
- IELTS 顶栏背景在 [css/heroui-bridge.css `--ielts-topbar-bg`](../css/heroui-bridge.css)
- three.js 背景在 [js/presentation/threeBackground.js](../js/presentation/threeBackground.js) 的 `forestGreenShader`

## 5. 题目相关维护

### 5.1 添加新题目(已存在的题型)

最简方式 → 使用 [tools/reading-editor.html](tools/reading-editor.html),按需选择题型组合,点击 `导出 ReadingExamSourceV1 JSON` 后:

1. 把导出的 JSON 包成 `.js`(套上 `(function registerReadingExamData...` 壳),放到 [assets/generated/reading-exams/](../assets/generated/reading-exams)
2. 在 [manifest.js](../assets/generated/reading-exams/manifest.js) 顶部新增一项:
   ```js
   "p1-custom-001": {
       "examId": "p1-custom-001",
       "dataKey": "p1-custom-001",
       "script": "./p1-custom-001.js",
       "title": "自定义示例题",
       "category": "P1"
   },
   ```
3. 设置 → 🔄 强制刷新题库

详细分步骤见 [03-导入新题型流程.md](03-导入新题型流程.md)。

### 5.2 修改已有题目的答案

打开对应 `.js` 文件,找到 `answerKey` 对象:

```js
"answerKey": {
    "q1": "viii",      // 原答案
    "q2": "iv",
    // ...
}
```

直接改值,保存,强制刷新即可。如果是多选/多答案题,`answerKey` 值可写成数组:

```js
"q14": ["coffee", "tea"]       // 任意顺序都算对
```

### 5.3 删除题目

1. 删除 `assets/generated/reading-exams/<id>.js`
2. 从 `manifest.js` 中移除对应条目
3. 强制刷新

> ⚠️ 已有该题的练习记录里 `examId` 不变,系统会标记为"未知题目"。如果要彻底清理,得在记录列表里手动删除,或通过 `📥 导入数据` 替换。

## 6. 日志与调试

### 6.1 浏览器 Console

- `[App]` 开头 → 主控制器
- `[Enhancer]` → 题目页注入脚本
- `[MarkdownExporter]` / `[PdfExporter]` → 导出器
- `[SHUI ...]` → 旧前缀,现统一为 `[IELTS CBT ...]`

### 6.2 启动期排错

按 F12 打开 DevTools,刷新页面,观察 Network 标签:

| 现象 | 排查路径 |
| ---- | -------- |
| `manifest.js` 404 | 部署路径错误,或 HTTP 服务器没指向项目根 |
| 单题 `.js` 404 | manifest 中的 `script` 路径与实际文件不一致 |
| `reading_exam_registry_missing` 报错 | 入口脚本顺序错乱,确保 `data/index.js` 先加载 |
| `practice-page-enhancer.js` 注入失败 | 浏览器拦截了 `postMessage`,关掉隐私插件或换 Chrome |

### 6.3 答案不准时

90% 是大小写、标点、连字符变体问题。打开 DevTools Console:

```js
// 调出归一化函数
window.AnswerMatchCore.normalizeToken('TRUE')       // → 'true'
window.AnswerMatchCore.normalizeToken('A. coffee')  // → 'A'
window.AnswerMatchCore.normalizeToken('café')       // → 'café'(原样,unicode 保留)
window.AnswerMatchCore.compareAnswers('Yes', 'TRUE') // → true(同义判定)
```

如果用户输入正确但被判错,检查归一化结果是否一致。

## 7. 性能调优

| 现象 | 建议 |
| ---- | ---- |
| 启动慢(> 5s) | 检查 `assets/generated/reading-exams/` 是否被全部 import → 不应,应按 manifest 懒加载 |
| 列表滚动卡顿 | `practice-history-list` 用了 grid,记录 > 500 时考虑虚拟滚动 |
| 内存占用高 | 关掉后台多余标签;DevTools → Memory → Heap snapshot 看是否有泄漏 |
| 移动端崩溃 | 关掉 three.js 背景:`localStorage.setItem('three_bg_disabled', '1')` 并刷新(需要在 `threeBackground.js start()` 加判定) |

## 8. 升级与版本管理

### 8.1 升级前

1. 全量备份:`📤 导出数据`(每个客户端各导一份)
2. 备份本地代码:`xcopy /e /i 网页版机考部署 网页版机考部署.bak.20260519`
3. 记录当前版本(README 顶部)

### 8.2 升级步骤

1. 拉新代码或替换文件
2. 比对 schemaVersion:
   - 旧 `ReadingExamSourceV1` 与新版本不一致时,需运行 schema 迁移(见 `js/data/repositories/`)
3. 启动一次,看 Console 是否有 `WARNING` / `ERROR`
4. 抽测 5 个不同题型的题目,确认能进入、提交、阅卷正常

### 8.3 回滚

把第 8.1 步备份的 `.bak` 目录替换回去,在每台客户端导入对应的 JSON 备份即可。

## 9. 故障排除清单

| 症状 | 可能原因 | 处理 |
| ---- | -------- | ---- |
| 启动屏一直转圈 | 主入口 JS 报错 | F12 Console 看红字 |
| 题目能打开但提交报错 | enhancer.js 没注入成功 | 检查同源策略,关掉浏览器隐私扩展 |
| 答案统计不对 | answerKey 拼写错或归一化差异 | 用 `AnswerMatchCore.normalizeToken` 验证 |
| 导出 PDF 空白 | 浏览器拦截弹窗 | 设置 → 弹出窗口 → 允许该站点 |
| 导出 PNG 失败 | 答题页含跨域图片 | 仅本地资源,无跨域。如确有外链,把图片改为 base64 嵌入 |
| 局域网客户端打不开 | 防火墙 | 服务器入站规则放行 80 / 8080 |
| 局域网客户端打开慢 | 第一次访问下载 ~3MB,无影响 | 加 nginx `gzip on` 提速 |
